Description

For the thesis work we developped the "Dungeon Tool": a support tool for the role-play game Dungeons&Dragons. The DungeonTool makes more easy the game experiance and it gives to the players the basic elements to play D&D.
Programming language :

Tools

Features

Multiplayer and architecture.

We used the Networking of Unity for the managment of the network. It is done with the component Network Manager.

Architecture client / server where the DM is the host and the PC are the clients.

Game session

The Dungeon Tool is a multiplayer LAN game. The host is the DM and he create a new game session where clients(PC) can connect to it. All the palyers have the same view of the world, every changes can be seen in real time. Player disconnection: the character stays in the scene. If the PC reconnects, he can have again the controll of his character. End game: that is decided by the DM.

Game sructure

Every players have their own character PC(Player Character) and all players will be spawned in a fantasy world created by the DM(Dungeon Master). The DM shall decided the rules and he controls the game. The player can create his own character in the below form: Each players have a turn to play and it is handled by the system.

Rules and GamePlay

The rules are the same of D&D. The DM handle everything, infact he have the control of the game, for instance, he can move the characters of the players into the map.

Design

There are 3 main scenes with simple and user-friendly interface. Below there are some descriptions:

The image of the dragon was designed with GIMP. At the top right there is the title of the game (Dungeon Tool) and under there is the address IP of the machine. The user shall chose between two type of characters

  • DM - Dungeon Master
  • PC - Player Character

The user shall chose between two type of characters; after the corresponding menu will be shown.

At the left there is the Operations menu with:

  • Add
  • Remove
  • Load
  • Save

Down in the screen there is the tile menu with the objects that can be scattered into the mapp:

  • Ground
  • Character
  • Item
  • Weapon
  • Armor

When you click an object, the menu of the object description shall open at the top right and you can modify the properties.

Scene of the game DM side. It is different than the PC view because the DM can move the view of the world. At the left there are the button -"Next Round", it can be used to end the turn of the current player and start the next turn. -"Save World" it can be used to save the current data.

On the PC view, if the character wasn't created appears the form for the character's creation, or else the character will load.
